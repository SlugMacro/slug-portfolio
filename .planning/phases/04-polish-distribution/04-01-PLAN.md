Create the `src/seo/` directory with three files:

**1. `src/seo/siteMetadata.ts`** — Centralized SEO constants:
```typescript
export const SITE_NAME = "Slug Macro";
export const SITE_DESCRIPTION = "Product Designer focused on trading systems and AI-native products.";
export const SITE_URL = "https://slugmacro.com"; // TODO: Update with actual production domain before deployment
export const DEFAULT_OG_IMAGE = `${SITE_URL}/og-default.png`; // TODO: Add og-default.png (1200x630) to public/ before deployment

export function pageTitle(page?: string): string {
  return page ? `${page} | ${SITE_NAME}` : `${SITE_NAME} | Product Designer`;
}

export function ogImageUrl(path?: string): string {
  return path ? `${SITE_URL}${path}` : DEFAULT_OG_IMAGE;
}
```

**2. `src/seo/pageMetaMap.ts`** — Route-to-metadata mapping for vite-plugin-react-meta-map. Export `pages` array of `PageMetaData` objects. Include:
- Static pages: index (`/`), about (`/about`), contact (`/contact`)
- Case study pages: one entry per project slug (whales-market, whales-predict, mention-network, geo-report)
- Each entry has: `url` (e.g., `"index.html"`, `"about/index.html"`, `"case-studies/whales-market/index.html"`), `bundleEntryPoint` (`"/src/main.tsx"`), `title`, `description`, `ogImage` (use `DEFAULT_OG_IMAGE`), `ogType` (`"website"` for static, `"article"` for case studies), `ogUrl` (absolute URL for the route).
- Import constants from `./siteMetadata`.
- Follow the exact code examples from the research document (04-RESEARCH.md "Building pageMetaMap from Content Loader Data" section).

**3. `src/seo/PageTemplate.tsx`** — HTML template component that vite-plugin-react-meta-map uses to generate per-route HTML files:
- Receives `PageMetaData` props
- Renders full `<html>` document with: charset, viewport, title, meta description, favicon link (`/favicon.svg`), OG tags (og:title, og:description, og:image, og:type, og:url), Twitter card tags (summary_large_image), `<div id="root"></div>`, and the bundle entry point script.
- Note: The `bundleEntryPoint` in pageMetaMap may be used by the plugin to inject the correct script. Follow the plugin's expected template structure. Include `<div id="root"></div>` as required.

**4. `public/favicon.svg`** — Create a simple SVG favicon. Use a minimal design: a rounded square with the accent color `#4c48ff` background and white "SM" initials text centered. Keep it under 20 lines of SVG. Example:
```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">
  <rect width="32" height="32" rx="6" fill="#4c48ff"/>
  <text x="16" y="22" text-anchor="middle" font-family="system-ui,sans-serif" font-weight="700" font-size="14" fill="white">SM</text>
</svg>
```

**5. `index.html`** — Update the existing file:
- Add `<link rel="icon" href="/favicon.svg" type="image/svg+xml" />` inside `<head>`
- Add `<meta name="description" content="Product Designer focused on trading systems and AI-native products." />` inside `<head>`
- Keep existing title tag (`<title>Slug Macro</title>`) — React 19 will override it at runtime

Do NOT create `public/og-default.png` — that requires an actual image asset the user will provide. The TODO comment in `siteMetadata.ts` covers this.

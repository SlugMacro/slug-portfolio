**Step 1: Install the plugin**
```bash
pnpm add -D vite-plugin-react-meta-map
```

**Step 2: Update vite.config.ts**
Add the meta map plugin to the Vite plugins array. Import it and configure with file paths pointing to the SEO files created in Plan 01:

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import tailwindcss from "@tailwindcss/vite";
import metaMapPlugin from "vite-plugin-react-meta-map";
import path from "path";

export default defineConfig({
  plugins: [
    react(),
    tailwindcss(),
    metaMapPlugin({
      pageMetaMapFilePath: "./src/seo/pageMetaMap.ts",
      pageTemplateFilePath: "./src/seo/PageTemplate.tsx",
    }),
  ],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
});
```

Note on plugin compatibility: The research flagged that `vite-plugin-react-meta-map` has not been explicitly tested with Vite 7. If the plugin fails to install or the build fails:
- **Fallback A:** Check the plugin's npm page for a newer version that supports Vite 7.
- **Fallback B:** Write a simple custom Vite plugin using the `generateBundle` hook that reads `pageMetaMap.ts`, renders the HTML template for each entry, and emits the files. This is ~40 lines of code.
- **Fallback C:** Write a post-build Node script (`scripts/generate-og-html.mjs`) that reads `dist/index.html`, injects route-specific meta tags, and writes copies to the appropriate subdirectories.

If any fallback is used, document which approach was taken and why in the SUMMARY.

**Step 3: Run production build**
```bash
pnpm build
```

**Step 4: Verify build output**
Check that per-route HTML files exist and contain OG tags:

Each file must contain: `og:title`, `og:description`, `og:image`, `og:type`, `twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`, plus the favicon link and script tag for the React app bundle.

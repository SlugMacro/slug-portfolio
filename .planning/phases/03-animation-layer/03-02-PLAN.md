   - `viewport={{ once: true, amount: 0.05 }}`

   ```tsx
     variants={staggerContainer}
     initial="hidden"
     whileInView="visible"
     viewport={{ once: true, amount: 0.05 }}
   >
     {leftCol.map((project) => (
       <motion.div key={project.meta.slug} variants={cardVariants}>
         <ProjectCard
           project={project.meta}
           onClick={() => onProjectClick(project.meta.slug)}
         />
       </motion.div>
     ))}
   </motion.div>
   ```

   Apply the same pattern to the right column (different className: `border-t border-border` without mt or border-l).

3. **Mobile column (the `flex flex-col gap-0 lg:hidden` section):** Apply the same stagger pattern. Wrap the mobile column div in `motion.div` with `staggerContainer` variants and `whileInView="visible"`, `viewport={{ once: true, amount: 0.05 }}`. Wrap each card in `motion.div` with `cardVariants`.

4. **Add `isolation: isolate` to the grid container.** On the outermost div (the one with `relative z-10 w-full px-6 pb-24 lg:ml-[25%] lg:w-[75%] lg:px-0`), add `isolation-isolate` to the className. Important: Use `viewport={{ once: true, amount: 0.05 }}`. The `once: true` prevents replay on scroll-back. The `amount: 0.05` triggers animation when just 5% of the container is visible, which feels responsive without waiting too long.

<task type="auto">
  <name>Task 2: Add hero parallax effect using useScroll and useTransform</name>
  <files>src/components/home/Hero.tsx</files>
  <action>
Modify `src/components/home/Hero.tsx`:

1. Add imports: `import { useScroll, useTransform, motion } from "motion/react"`.

2. Inside the Hero component function, add scroll tracking:
   ```tsx
   const { scrollY } = useScroll();
   const heroY = useTransform(scrollY, [0, 600], [0, -100]);
   const heroOpacity = useTransform(scrollY, [0, 400], [1, 0.3]);
   ```

   `useScroll()` with no arguments tracks the page scroll position. `useTransform` maps the scroll range to translateY and opacity values. As the user scrolls 0-600px, the hero text drifts upward by 100px (slower than the page = parallax). The text also fades to 0.3 opacity over the first 400px of scroll.

3. Convert the **desktop fixed hero div** from a plain `div` to `motion.div` and apply the parallax styles:
   ```tsx
   <motion.div
     className="fixed top-0 left-0 z-[1] hidden max-w-[1800px] items-start px-6 pt-[180px] lg:flex"
     style={{ width: "min(1200px, 56%)", y: heroY, opacity: heroOpacity }}
   >
     <h1 className="text-[clamp(56px,5.5vw,88px)] leading-[1.05] tracking-[-0.16vw] font-medium text-text-primary">
       Hi, I'm Slug Macro — a product builder designing and building complex systems, from trading infrastructure to AI-native products.
     </h1>
   </motion.div>
   ```

   Note: The `style` prop merges the existing `width` with the motion values `y` and `heroOpacity`. Framer Motion handles `y` as a `MotionValue` applied to `translateY` on the transform property. The `opacity` MotionValue overrides any CSS opacity.

4. **Leave the mobile hero completely unchanged.** No parallax on mobile/tablet -- the mobile hero is a static in-flow element (`lg:hidden`), not fixed. Parallax on a scrolling-in-flow element would feel broken.

5. **Leave the desktop spacer unchanged** (`hidden lg:block`, height 70vh).

Why these values: The `[0, 600]` to `[0, -100]` mapping means for every 6px of scroll, the hero moves 1px upward. This is subtle enough to feel like depth without being disorienting. The opacity fade prevents the hero text from being fully visible behind the grid cards (which scroll over it). These values are tunable -- start here and adjust if needed.
